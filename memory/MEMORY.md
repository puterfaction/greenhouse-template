# Persistent Memory — index

> **This is the index of the durable memory layer.** It loads into context each
> session. Keep it to *one line per memory file* — a title, a link, and a hook — plus
> the short standing sections below. Never put memory *content* here; the content lives
> in the per-topic files this index points to. `/tend` keeps it pruned (soft limit ~300
> lines).
>
> Each per-topic file is one fact or one project, with frontmatter naming its `type`
> (`user`, `feedback`, `project`, or `reference`). See `example-project.md` and
> `ship-threshold-example.md` for the shape. Link between memories with `[[file-name]]`.

## About the person
- *One or two lines of the most load-bearing facts — who they are, what they're working
  toward. The full picture lives in [[who-we-are]]; this is just the pointer.*

## Technical preferences
- *Editor/browser/tooling preferences, git identity, "simple over clever," etc.*

## Patterns that work
- *Approaches that have repeatedly worked. Consult before repeating a task type; add
  lessons after. E.g. "Test immediately after building, report specific errors."*

## Patterns that don't work
- *Dead ends worth remembering so they aren't retried. E.g. "Library X version 1.2
  crashes; 1.4 works." Memory that only accretes is a pattern that doesn't work — prune.*
