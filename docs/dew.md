# Dew — the morning brief

Dew is the layer that reaches *out*. Everything else in this kit waits for you to open a
session; dew forms fresh every morning and lands on your phone: what's alive, what's
next, and one small ask — generated by an agent that read your `now.md` while you slept.

The name is the metaphor: it appears on the leaves each morning, automatically, and it's
gone by evening. The archive (`dew/YYYY-MM-DD.md`, one file per day) is the record; the
push notification is the product.

## Why it works

- **It closes the loop on `now.md`.** The "what's alive" file is only useful if it gets
  read. Dew reads it every morning and hands you the one-screen version, so tending
  now.md pays off even on days you never open a laptop.
- **The commit is the proof-of-life.** Every run commits its brief to the repo. A missing
  `dew <date>` commit by ten past the hour means the run failed; a committed brief with
  `pushed: false` means generation worked but the phone push didn't. You can verify the
  system from any device with repo access — no dashboards.
- **Push is best-effort, the archive is the record.** ntfy going down loses you a
  notification, never a brief.

## Setup

1. **Copy the config:** `cp dew/config.example.json dew/config.json`, set your `timezone`.
2. **Phone push (optional but the point):** install the [ntfy](https://ntfy.sh) app
   (free, open source), subscribe to a topic with a long random name (the topic name is
   the only secret — treat it like a password, e.g. `dew-a1b2c3d4e5f6`), and put it in
   `config.json` as `ntfy_topic`. Leave it empty to run archive-only.
   **If your repo is public, don't commit a real topic name** — anyone who reads it can
   send you (or read your) pushes. Commit the config only in private repos; in public
   ones, keep `dew/config.json` untracked and use a CI secret or archive-only mode.
3. **Schedule a runner.** The behavior lives entirely in `dew/routine-prompt.md` — any
   runner just tells an agent to follow that file. Options, in order of least plumbing:
   - **Claude Code cloud routine** (works even when your machine is off): ask Claude Code
     to "schedule a daily routine at <time> that follows the instructions in
     dew/routine-prompt.md exactly, from the repo root." The routine runs against your
     GitHub repo in the cloud. Note cron is UTC-fixed — your local send time shifts an
     hour when DST changes.
   - **Local cron/launchd + headless CLI** (machine must be awake at send time — poor fit
     for laptops): schedule `claude -p "Follow the instructions in dew/routine-prompt.md
     exactly, from the repo root."` in the repo directory.
   - **CI cron** (GitHub Actions or similar): a scheduled workflow that checks out the
     repo and runs the same headless command with an API key. Most plumbing, most
     reliable; the fallback if the others fail you.
4. **Verify the first unattended run** the way the system verifies itself: next morning,
   check for the `dew <date>` commit and `pushed: true` in its frontmatter.

## Customizing the voice

Edit `dew/routine-prompt.md` — the runner just follows it, so voice, bullet kinds, length
cap, and rules are all one-file edits with no re-scheduling. The prompt reads
`who-we-are.md` first, so the brief's register follows whatever you've recorded there.
The example (`dew/2026-01-03-example.md`) shows the default shape: one-line greeting,
labeled bullets (`Alive:` / `Up next:` / `Small ask:`), no fixed counts — the brief
scales to the day.

## Design notes (learned in production)

- **Point the runner at the prompt file, never inline the prompt into the scheduler.**
  Scheduled-job configs are annoying to edit; files in the repo are one commit away.
- **The dedup rule needs the file filter.** "Read the recent briefs" must say *dated
  brief files* — a zero-context agent sorting `dew/` by mtime will grab `index.json`
  every time and the don't-repeat-yesterday's-ask rule silently breaks.
- **Send the push body from a file** (`--data-binary @file`), not an interpolated shell
  string — the first brief containing `"` or `$` will break or inject otherwise.
- **Idempotency matters more than it looks:** re-runs happen (manual triggers, scheduler
  double-fires). Overwrite-and-update beats append-and-duplicate.
